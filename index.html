    • Carl Schmertmann's R Code for Kannisto Extrapolation of Mortality Rates to Oldest Ages

      (mortality, R code)

      Described in the explanatory notes (2019) by Prof. Schmertmann, and in Thatcher, Kannisto, and Vaupel (1998), the Kannisto model of oldest-age mortality makes use of Kannisto's observation that a logit function can provide a good fit for oldest-age mortality rates in modern populations. This can be useful for extrapolating mortality to oldest ages. –February 2022 ^

      (migration, R package)

    • Raftery et al's BayesPop: Probabilistic Population Projection Software

      (population projections, fertility, migration, mortality, R package)

      From the project's About page: "We are developing a statistical framework for probabilistic projection and estimation of population, fertility, mortality and migration for all countries, in collaboration with the United Nations Population Division.     • Tom Wilson's Model Migration Schedule Fitting Example

      (migration, Excel spreadsheet)

Tom Wilson of the University of Melbourne has extended and improved the Rogers-Castro migration model, offering both a student curve and an alternative post-retirement curve, and linked above is a related Excel workbook to apply the model.     • Eddie's Hamilton-Perry Projection with Components and Stable Population Information Code*

      (population projections, R code, Excel spreadsheet)

      The Hamilton-Perry method, which uses ratios of change in cohort sizes between two points in time (cohort change ratios) to project population, is a useful population analysis resource, but the core method doesn't include information on the components of population change. This R code for population projections employs cohort change ratios with optional inputs for fertility (iTFR – Hauer and Schmertmann (2019) and Hauer, Baker, and Brown (2013)), mortality (Brass' relational logit model – information here and here), and net migration (information here, here, and here) as well as gross migration, and also returns estimates of stable population growth and age structure. To make the code work, you should be able to simply select-all, copy, and paste into an R command line (the example input data is linked through the Internet, and can be simply downloaded for review). Here is a Shiny for R application for the code. And more information, including formulas, a spreadsheet demonstration to replicate results, slides for a related talk, as well as R code with input files for several examples, including an errors review version, a single-year-of-age version, and a stochastic version, is all available in a related GitHub repository. –May 2020 (updated January 2022) ^

    • Hauer and Byar's IRS County-to-County Migration Data, 1990 to 2010, Replication Code

      The related paper's abstract explains:
      "
      Background: The county-to-county migration data of the Internal Revenue Service's (IRS) is an incredible resource for understanding migration in the United States. Produced annually since 1990 in conjunction with the US Census Bureau, the IRS migration data represents 95% to 98% of the tax-ﬁling universe and their dependents, making the IRS migration data one of the largest sources of migration data. However, any analysis using the IRS migration data must process at least seven legacy formats of this public data across more than 2000 data files - a serious burden for migration scholars.
      Objective: To produce a single, flat data file containing complete county-to-county IRS migration flow data and to make the computer code to process the migration data freely available.
      Methods: This paper uses R to process more than 2,000 IRS migration ﬁles into a single, flat data file for use in migration research.
      Contribution: To encourage and facilitate the use of this data, we provide a single, standardized, flat data file containing county-to-county one-year migration flow for the period 1990-2010 (containing 163,883 dyadic county pairs resulting in 3.2 million county-year observations totaling over 343 million migrants) and provide the full R script to download, process, and flatten the IRS migration data.
      "
      A very neat and especially valuable work, by Matt Hauer of Florida State University and James Byars of the University of Georgia. –September 2019 (updated February 2022) ^

    • US Department of Housing and Urban Development's Zip Code Crosswalk Files

      (geographic, data and materials, web interface)

      From the home page: "One of the many challenges that social science researchers and practitioners face is the difficulty of relating United States Postal Service (USPS) ZIP codes to Census Bureau geographies. There are valuable data available only at the ZIP code level that, when combined with demographic data tabulated at various Census geography levels, could open up new avenues of exploration. "While some acceptable methods of combining ZIP codes and Census geography exist, they have limitations. To provide additional avenues for merging these data, PD&R has released the HUD-USPS Crosswalk Files. These unique files are derived from data in the quarterly USPS Vacancy Data.     • Eddie's Brass Relational Logit Mortality Model Code*

      (mortality, R code)

      I think William Brass' most famous contribution to demography (he had many, and is surely among the most productive and revered demographers ever) is the Brass Relational Logit Model for mortality. The Brass Relational Logit Model is used to simply and reasonably adjust the level and shape of a life table's lx (survivorship) curve, based on the level and shape of another life table's lx curve. This ability for a reasonable adjustment by two parameters is useful for many things, including (1) life table estimation with limited data, where a life table with very large margins of error and an unreliable pattern across age can be modeled with a life table that has very small margins of error, and (2) life table forecasting, in which the the level and shape terms (alpha and beta, respectively) can be projected over time. Brass alpha in particular has been a subject of interest for mortality forecasting, as it is believed to change linearly over time for many cases/periods (note rectangularlization though) (Brass beta is often trendless-- I think particularly in developed regions).
