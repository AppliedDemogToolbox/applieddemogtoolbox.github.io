To make the code work, you should be able to simply select-all, copy, and paste into an R command line (the example input data is linked through the Internet, and can be simply downloaded for review). Here is a Shiny for R application for the code.     • R Code for Fitting the Rogers-Castro Multi-Exponential Model Migration Schedule* (migration, R code)

      ⧉ R Code for Fitting the Rogers-Castro Multi-Exponential Model Migration Schedule
      Model migration age schedules have been developed and described over the past few decades (a central resource: Rogers and Castro, 1981), and they are useful for understanding human migration, which has some recurrent patterns for childhood, labor force ages, retirement ages, and old ages. Tom Wilson of the University of Melbourne has extended and improved-upon the Rogers-Castro migration model, adding both a student curve and alternative post-retirement curve, and also provided terrific guidance and instruction on the overall model. This R code uses the guidance and instruction from Dr. Wilson's model migration schedule articles, and their related Excel workbooks, along with R's random number generator, and sampling (Monte Carlo) from recursively smaller uniform distributions of model parameters to locate a best fit for each of the components. The sampling-based fitting process is just a "DIY" solution (I haven't been able to find a reference for fitting-by-sampling done this way, or an evaluation of it - please let me know if you know one, or if you have insights that might be instructive or generally helpful), and it hasn't been carefully tested at all, but it's neat it appears so far to do the job pretty well. When I developed the code, I thought that using uncertain bounds as the parameters' starting inputs for fitting Rogers-Castro was novel, but soon after I learned of a neat and helpful R package developed months before this code that uses uncertain bounds, indeed uniform distributions, for these: migraR, by J. Sebastian Ruiz-Santacruz and Jackson Garcés Hernández, also listed on this page. I think relying on uniform distribution bounds rather than single values for starting-parameter selection can make the overall process of fitting the model easier. To make the code work, you should be able to simply select-all, copy, and paste into an R command line (the example input data is linked through the Internet, and can be simply downloaded for review). Copies of the code with some application of and comparison to R's nls() and lm() functions, and to more plain Monte Carlo, are available here. Here is a Shiny for R interface with linked code to view the model fitted to American Community Survey Public Use Microdata Sample data (accessed via IPUMS) for US states and California counties. - August 2018 (updated July 2025)

      ⧉ Brass Relational Logit Mortality Model Code
      I think William Brass' most famous contribution to demography (he had many, and is surely among the most productive and revered demographers ever) is the Brass Relational Logit Model for mortality. Brass alpha in particular has been a subject of interest for mortality forecasting. Linked above is an R function/script to apply the Brass Relational Logit model, and make selected output graphics.
